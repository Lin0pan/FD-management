## Codebase Patterns
(Carried forward from the US-14 batch, archived at
`scripts/ralph/archive/2026-07-22-us-14-configure-business-rules/progress.txt` — read that file for
the full story-by-story history.)

- Typed domain errors live in `src/domain/errors.ts`: extend the abstract `DomainError` base class,
  declare `readonly code` from the `DomainErrorCode` union, and carry the offending values as
  fields. Never `throw new Error("…")`. Add the new kind to the union when adding a class.
- Domain modules are validated-on-construction: a `createX(input)` factory throws naming the field,
  so no partially-valid object can exist. Prefer this over scattering checks at call sites.
- Coverage on `src/domain` + `src/application` is gated at 100% including **branches**. A
  short-circuiting guard needs a test per operand — budget one test per clause or the gate fails.
- Use a type guard (`function isX(v: number): v is X`) rather than an `as` cast to narrow a
  validated primitive into a literal-union type — the repo standard forbids `any` and `!`.
- Test helpers take a `Partial<Input>` overrides object over a valid baseline, so each test names
  exactly the one rule it is about (see `settings.test.ts`).
- Use cases live in `src/application/<area>/<verb-noun>.ts`, one per business action, each taking a
  `Deps` object of ports as its first argument and its `Input` as the second. Ports are declared in
  `src/application/ports.ts` and stay type-only.
- Application tests keep their hand-written fakes **inside the test file** — a separate `fakes.ts`
  under `src/application/` would be pulled into the 100% coverage gate.
- Every state change appends an `AuditEntry` (`what` / `changedFields` / `when` / `why`, never an
  actor); `when` comes from `clock.now()`, not from any business date in the input.
- Run `npx prettier --write <paths>` before `npm run lint`; the pre-commit hook formats anyway, but
  it rewrites files mid-run otherwise.
- Local quality gate: `npm run lint && npm run typecheck && npm run test:coverage && npm run build`.
- Infrastructure adapters live in `src/infrastructure/prisma/<thing>-repository.ts` and implement a
  port from `src/application/ports.ts`. Rows re-enter the domain through the `createX` factory, so a
  hand-edited database cannot bypass an invariant.
- Integration tests migrate a **throwaway** SQLite file: `mkdtempSync` + `execFileSync("npx",
  ["prisma","migrate","deploy"], { env: { ...process.env, DATABASE_URL: url } })` in `beforeAll`
  (60s timeout), `new PrismaClient({ datasourceUrl: url })`, `rmSync` in `afterAll`. Use an
  **absolute** `file:` path — the generated client resolves relative paths against the cwd, while
  the CLI resolves them against `prisma/`.
- SQLite has no enum type: store a union as `String` and narrow it back with a domain parser
  (`parseWeekColour`). Parsing belongs in the domain, not the adapter.
- A `"use server"` module may export **nothing but async functions** — every export becomes a
  callable server endpoint. Put form-state objects and types in a sibling module. `npm run build`
  does **not** catch this; it fails at page load, so a route needs an actual browser/E2E hit.
- A route directory carries its own `deps.ts` composition root — the one place real adapters are
  chosen. The page hands that object to a use case and does nothing else with it.
- German text for a rejected field comes from a keyed map in `src/i18n/de.ts` (e.g.
  `de.settings.errorFields`), keyed by the `field` string the domain error carries.
- Server components that read data changed through a form need `export const dynamic =
  "force-dynamic"`, or Next prerenders them at build time and the screen shows stale values.
- Playwright specs live in `tests/e2e/*.spec.ts`, may import `@/…`, and take their expected text
  from `src/i18n/de.ts` rather than hard-coding German. Specs sharing `data/e2e.db` must declare
  `test.describe.configure({ mode: "serial" })`. Address form fields by `id`, not `getByLabel`.
- **No browser MCP tooling is configured.** For a UI story, verify with a throwaway Playwright run
  against the built app plus a curl of the rendered HTML — that is what caught the `"use server"`
  export bug four green gates missed.
- **Faker is a devDependency** as of US-004 (`@faker-js/faker`). Call `faker.seed(…)` once at the top
  of a spec so a failure is reproducible, and keep fixed dates literal — only names and addresses
  come from Faker.
- Prisma cannot express a **partial/filtered unique index**. Where one is needed, hand-write the
  `CREATE UNIQUE INDEX … WHERE …` at the end of the migration SQL and say so in a `schema.prisma`
  comment — regenerating the migration silently drops it. Pre-release the whole history is
  regenerated (`rm -rf prisma/migrations` + `prisma migrate dev --name init --create-only`), which is
  exactly when that happens. `migrate dev` is interactive: delete `data/fd.db` first or it stops.
- A **nested Prisma `create`** (customer + members + certificate + card in one `data`) is already one
  transaction — no `$transaction` wrapper needed for "all or nothing".
- Translate a Prisma `P2002` into the matching typed domain error in the adapter, checking the
  constraint target so a *different* future constraint surfaces as itself rather than as a lie.
- **Check `git status` before starting a story.** A killed iteration leaves finished but uncommitted
  work behind while `prd.json` still reads `passes: false`. Review the diff before writing anything.
- `eslint`'s `react-hooks/error-boundaries` forbids constructing JSX inside a `try` — the `catch`
  could never fire, because React renders after the function returned. Await into a variable inside
  the `try`, build the JSX after it. Only lint catches this, not `npm run build`.
- Next 16 gives a dynamic route its `params` as a **`Promise`** — `await` it in the page.
- Where a screen must render whether or not a rule succeeds, give the rule a **total** form beside
  the throwing one (`findLowestFreeNumber` / `lowestFreeNumber`) and write the throwing one in terms
  of it — two entry points, one statement of the rule.
- Dates cross the form boundary as **UTC calendar days**: `<input type="date">` gives `YYYY-MM-DD`
  and the adapter pins it to `T00:00:00.000Z`, matching how the domain compares birthdates. Parsing
  in local time lands a date typed in Germany on the day before.
- An E2E spec whose data lives in the shared `data/e2e.db` must be **serial** and must not assume a
  clean register: it shares the customer-number sequence with every other spec in the file. Read the
  proposed number off the screen and assert relative to it rather than hard-coding an id, and prove
  "nothing was written" by reloading the form and seeing the same number still free.
- A record that the domain owns is built by a `createX(input, today)` factory in `src/domain/…`; the
  use case above it never validates fields itself. Where a rule already derives something
  (`composition`), call it for its **rejections** and discard the result rather than adding a second
  check — one place states the rule.

    story free of the Faker dependency (birthdates are not personal data).

    `distribution/` still are.

    partial-unique-index violation (US-004/US-005), and waiting-list promotion (US-12). The

    `registration.spec.ts` run in *different workers* (`fullyParallel`), so a settings spec asserting
    on the customer count would race the registrations. Enabling it means either moving it into the
    registration spec or making the whole run serial — a decision for the story that needs it.
  - CI runs with `retries: 2`. A retried registration spec restarts against a database the first
    attempt already wrote to, so the happy path would see number 2. If that ever flakes, the answer
    is to stop asserting the *absolute* proposed number ("1") and only compare before/after.
